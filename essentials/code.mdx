DubStack provides excellent syntax highlighting for code examples. This guide shows how to display code inline and in dedicated blocks.

## Inline code

To denote a `word` or `phrase` as code, enclose it in backticks (\`).

```markdown
To denote a `word` or `phrase` as code, enclose it in backticks (`).
```

## Code blocks

Use [fenced code blocks](https://www.markdownguide.org/extended-syntax/#fenced-code-blocks) by enclosing code in three backticks. Add the programming language after the opening backticks for syntax highlighting.

```javascript
const config = {
  apiUrl: 'https://api.dubstack.com',
  apiKey: process.env.DUBSTACK_API_KEY
};
```

````markdown
```javascript
const config = {
  apiUrl: 'https://api.dubstack.com',
  apiKey: process.env.DUBSTACK_API_KEY
};
```
````

## Adding filenames

Optionally add a filename after the language for better context:

```javascript config.js
const config = {
  apiUrl: 'https://api.dubstack.com',

};
```

````markdown
```javascript config.js
const config = {

  apiKey: process.env.DUBSTACK_API_KEY
````


## Line highlighting

Highlight specific lines to draw attention to important code:

```javascript {2,4-6}
const config = {
  apiUrl: 'https://api.dubstack.com',  // Highlighted
  timeout: 5000,
  headers: {                            // Highlighted
    'Authorization': `Bearer ${token}`  // Highlighted
  }                                     // Highlighted
};
```

````markdown

const config = {
````

## Code annotations

Add inline comments to explain complex code:

```typescript
// Initialize DubStack client
const client = new DubStack({
  apiKey: process.env.DUBSTACK_API_KEY,  // Get from dashboard
  baseUrl: 'https://api.dubstack.com',   // Production URL
  timeout: 30000                         // 30 second timeout
});

// Fetch documentation
const docs = await client.docs.list();
```

## Best practices

<Callout type="success">
**Do** include complete, runnable examples that users can copy and paste.
</Callout>

<Callout type="success">
**Do** use realistic data instead of placeholders like `foo`, `bar`, `example`.
</Callout>

<Callout type="warning">
**Don't** include real API keys or secrets in code examples.
</Callout>

<Callout type="warning">
**Don't** show incomplete code that won't run without modification.
</Callout>
