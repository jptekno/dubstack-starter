Claude Code is Anthropic's official CLI tool. This guide shows how to use Claude Code with the **DubStack MCP server** to manage your documentation directly from the command line.

## Prerequisites

- Active Claude subscription (Pro, Max, or API access)
- DubStack account with API access
- Node.js 18+ installed

## What is MCP?

[Model Context Protocol (MCP)](https://modelcontextprotocol.io) is an open standard that lets AI assistants like Claude securely access external tools and data sources. The DubStack MCP server lets Claude read and write your documentation, manage navigation, update themes, and more.

## Setup

### 1. Install Claude Code

```bash
npm install -g @anthropic-ai/claude-code
```

### 2. Install DubStack MCP Server

```bash
npm install -g @dubstack/mcp-server
```

### 3. Configure MCP in Claude Code

Add the DubStack MCP server to your Claude Code configuration:

```bash
claude config mcp add dubstack
```

When prompted, provide:
- **Server command**: `npx @dubstack/mcp-server`
- **Environment variables**:
  - `DUBSTACK_API_URL`: Your DubStack API endpoint
  - `DUBSTACK_API_KEY`: Your API key from dashboard

### 4. Verify Connection

```bash
claude mcp list
```

You should see `dubstack` in the list of available MCP servers.

## Using Claude Code with DubStack

Once configured, Claude can directly interact with your documentation:

<CodeGroup>
```bash Read documentation
# Ask Claude to summarize a doc
claude "What does our API authentication page say?"

# Claude will use MCP to fetch the content
```

```bash Update documentation
# Ask Claude to update a doc
claude "Update the quickstart guide to include the new CLI installation step"

# Claude will use MCP to write changes
```

```bash Create new pages
# Ask Claude to create documentation
claude "Create a new guide for deploying to Vercel"

# Claude will create the MDX file and update navigation
```
</CodeGroup>

## MCP Capabilities

The DubStack MCP server provides these tools to Claude:

<AccordionGroup>
<Accordion title="dubstack_read_doc - Read documentation content">
  Fetches content of any documentation page by path.

  **Example**: "Show me what's in `/api-reference/authentication`"
</Accordion>

<Accordion title="dubstack_write_doc - Create or update docs">
  Creates new documentation pages or updates existing ones.

  **Example**: "Add a troubleshooting section to the installation guide"
</Accordion>

<Accordion title="dubstack_list_docs - List all documentation">
  Returns a list of all documentation pages with their paths and titles.

  **Example**: "What documentation pages do we have?"
</Accordion>

<Accordion title="dubstack_update_nav - Update navigation">
  Modifies the documentation navigation structure in `dubstack.json`.

  **Example**: "Add the new API guides section to the sidebar"
</Accordion>

<Accordion title="dubstack_update_theme - Update theme settings">
  Changes theme colors, fonts, or layout settings.

  **Example**: "Change the primary color to #3b82f6"
</Accordion>

<Accordion title="dubstack_search - Search documentation">
  Searches across all documentation content.

  **Example**: "Find all pages that mention webhooks"
</Accordion>
</AccordionGroup>

## Create `CLAUDE.md`

Create a `CLAUDE.md` file at the root of your documentation repository to train Claude Code on your specific documentation standards:

````markdown
# DubStack Documentation

## Working relationship
- Push back on ideas when you have better suggestions - cite sources and explain reasoning
- ALWAYS ask for clarification rather than making assumptions
- NEVER lie, guess, or make up information

## Project context
- Format: MDX files with YAML frontmatter
- Config: dubstack.json for navigation, theme, settings
- Components: DubStack components (Card, Tabs, Accordion, Steps, etc.)
- MCP Server: Use DubStack MCP tools to read/write documentation

## Content strategy
- Document just enough for user success - not too much, not too little
- Prioritize accuracy and usability
- Make content evergreen when possible
- Search for existing information before adding new content
- Check existing patterns for consistency
- Start by making the smallest reasonable changes

## Frontmatter requirements
- title: Clear, descriptive page title
- description: Concise summary for SEO/navigation
- icon: (optional) Lucide icon name for nav

## Writing standards
- Second-person voice ("you")
- Prerequisites at start of procedural content
- Test all code examples before publishing
- Match style and formatting of existing pages
- Include both basic and advanced use cases
- Language tags on all code blocks
- Alt text on all images
- Relative paths for internal links

## Git workflow
- NEVER use --no-verify when committing
- Ask how to handle uncommitted changes before starting
- Create a new branch when no clear branch exists
- Commit frequently throughout development
- NEVER skip or disable pre-commit hooks

## Do not
- Skip frontmatter on any MDX file
- Use absolute URLs for internal links
- Include untested code examples
- Make assumptions - always ask for clarification
````

## Example workflows

### Document a new API endpoint

```bash
claude "We just added a POST /api/uploads endpoint. Document it with:
- Authentication requirements
- Request body schema (file: File, folder: string)
- Success response (201 with upload_id and url)
- Error responses (400, 401, 413)
Add it to the API Reference section"
```

### Update documentation for a breaking change

```bash
claude "We renamed the environment variable API_KEY to DUBSTACK_API_KEY.
Update all documentation pages that mention it and add a migration note"
```

### Generate documentation from code

```bash
claude "Read src/lib/theme.ts and create documentation for the theme
customization API. Include all available options and examples"
```

## Troubleshooting

<AccordionGroup>
<Accordion title="MCP server not found">
  Ensure `@dubstack/mcp-server` is installed globally:

  ```bash
  npm install -g @dubstack/mcp-server
  npx @dubstack/mcp-server --version
  ```
</Accordion>

<Accordion title="Authentication errors">
  Verify your API key is set correctly:

  ```bash
  claude config mcp get dubstack
  ```

  Get a new API key from your [DubStack dashboard](/admin/api-keys).
</Accordion>

<Accordion title="Changes not appearing">
  DubStack syncs documentation via Git. Ensure:

  - Your Git integration is configured
  - Changes are committed and pushed
  - Git webhook is working (check dashboard)
</Accordion>
</AccordionGroup>
